# Support

## Scope

This repository maintains a local, script-driven governance control plane. It does not provide a hosted control-plane service, uptime SLA, or on-call support commitment.

## Where to Get Help

1. Check `README.md` and `OPERATOR_LOOP_GUIDE.md` for entrypoints and operator flow.
2. Check `docs/runbook_ops.md` and `docs/loop_operating_contract.md` for operational and policy details.
3. Open a GitHub Issue for bugs, feature requests, or operator questions.

## Reporting Bugs

When opening a bug report, include:
- expected vs actual behavior,
- exact command(s) run,
- key output lines and exit code,
- environment details (`python --version`, OS).

## Security Issues

For security-sensitive findings, follow `SECURITY.md`.
`docs/security.md` remains the internal operations companion for maintainers and operators.
If public disclosure would create risk, avoid posting exploit details in a public issue and request a private contact path as described in `SECURITY.md`.

## Code of Conduct

Please follow `CODE_OF_CONDUCT.md` in all project interactions.
